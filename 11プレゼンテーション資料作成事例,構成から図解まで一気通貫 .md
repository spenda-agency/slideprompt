PowerPointのデザイン（藍色の文字、薄い青色の背景、著作権表記など）を再現しつつ、「テキストを書くだけで、このデザインのスライドが無限に生成される仕組み」 を構築しましょう。

この仕組みさえ作ってしまえば、今後はデザインで悩む時間がゼロになります。

全体像：今回の「仕組み化」フロー
今回は「デザイン（型）」と「内容（文字）」を完全に分離します。

【型】カスタムテーマ (style.css)：添付PPTのデザインルールをここに記述（1回作ればOK）。

【内容】原稿ファイル (slide.md)：毎回ここだけを書く。

【ロボット】自動変換 (convert.yml)：型と内容を合体させてPDFを作る。

手順1：【型】デザインを定義する (style.css)
まず、添付PPTの「藍色のフォント 」「薄い青色の背景 」「Copyright 」をコード（CSS）で定義します。



Cursor（またはVS Code）で、プロジェクトのルート（一番上の階層）に style.css というファイルを作り、以下をコピペしてください。

CSS

/* @theme spenda-style */

section {
  background-color: #ffffff;
  color: #333333; /* 地の文字は黒色 [cite: 5] */
  font-size: 30px;
  padding: 50px;
  font-family: 'Helvetica', 'Arial', sans-serif;
}

/* ヘッダー・フッター（著作権表記） */
footer {
  position: absolute;
  bottom: 20px;
  left: 50px;
  font-size: 14px;
  color: #666;
}

/* タイトルページのデザイン */
section.lead {
  background-color: #f0f8ff; /* 薄い青色の背景  */
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
  text-align: center;
}

/* 見出しのデザイン */
h1 {
  color: #1e3a8a; /* 重要で差別化したい藍色  */
  font-size: 180%;
  margin-bottom: 20px;
}

h2 {
  color: #1e3a8a; /* 藍色  */
  font-size: 150%;
  border-bottom: 2px solid #e0f2fe; /* 薄い青のライン */
  padding-bottom: 10px;
}

/* 強調（薄い青のハイライト） [cite: 6] */
strong {
  background: linear-gradient(transparent 60%, #e0f2fe 60%);
  font-weight: bold;
  padding: 0 4px;
}

手順3：【ロボット】変換レシピを更新する (convert.yml)
最後に、ロボットへの指示書（GitHub Actions）を少しだけ修正し、「自作のデザイン(style.css)を使ってね」 と伝えます。

.github/workflows/convert.yml を以下のように書き換えます。

YAML

name: デザイン適用済みスライド作成
on: push

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: PDFに変換 (カスタムテーマ適用)
        uses: marp-team/action-marp-cli@v2
        with:
          path: slide.md
          output: output.pdf
          theme: style.css  # ★ここが重要：自作デザインファイルを読み込む
          
      - name: 成果物を保存
        uses: actions/upload-artifact@v3
        with:
          name: presentation-pdf
          path: output.pdf
手順4：実行と確認
これら3つの変更（style.css, slide.md, convert.yml）を GitHub Desktop でCommit & Push します。

GitHubの [Actions] タブで処理が終わるのを待ちます。

生成されたPDFをダウンロードして確認してください。

添付のPPTに近い「藍色の文字」「薄い青のアクセント」が自動で適用されているはずです。

今後の運用：テーマを変えて使い回すには？
この仕組みの最大の利点は、「別のテーマ（企画書や報告書）」を作るときの手間が圧倒的に減ることです。

新しい資料を作りたい場合
slide.md をコピーして、例えば monthly_report.md（月次報告）という名前にします。中身のテキストを変えるだけで、デザインは勝手に統一されます。

デザインを一括変更したい場合（リブランディング）
style.css の「藍色 (#1e3a8a)」を「赤色」に書き換えてPushするだけで、過去の資料も含めて、次回生成時に全て新しいブランドカラーに自動更新されます。これがパワポにはできない、Git管理ならではの強みです。

まずは一度Pushしてみて、生成されたPDFの色味を確認してみてください。微調整（もう少し青を薄くしたい、など）が必要であれば、CSSの数値をいじるだけで即座に修正可能です。
